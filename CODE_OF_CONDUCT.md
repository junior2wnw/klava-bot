# Code of Conduct

## Our Pledge

We want Klava to be a serious public project where people can collaborate without noise, hostility, or status games.

Contributors, maintainers, and users are expected to make participation respectful, technically focused, and constructive for everyone.

## Our Standards

Examples of behavior that help:

- giving technical feedback clearly and calmly;
- assuming good intent while still holding a high engineering bar;
- sharing context, evidence, and reproduction details;
- helping other contributors understand tradeoffs instead of posturing.

Examples of unacceptable behavior:

- harassment, insults, or personal attacks;
- bigotry or discriminatory language;
- doxxing or exposing private information;
- deliberately disruptive trolling;
- repeated bad-faith technical argument after maintainers have made a decision.

## Scope

This Code of Conduct applies in:

- GitHub issues;
- pull requests;
- discussions;
- project chat spaces;
- release and contribution discussions tied to this project.

## Enforcement

Maintainers may remove, edit, or reject comments, issues, PRs, or contributions that violate this Code of Conduct.

Repeated or severe violations may result in temporary or permanent bans from the project spaces.

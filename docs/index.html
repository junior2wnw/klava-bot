A desktop agent for local tasks, shell work, and controlled machine changes.

What makes it different

Built for local work, not just chat.

Klava is meant to keep local work in one place: inspect the machine, ask for approval, execute a bounded action, and leave behind records another engineer can review.

Upstream lineage

Fork lineage, documented plainly.

Klava grows from OpenClaw. The repo documents that relationship directly instead of relying on a missing fork badge or vague wording.

Planned workflows

Examples of the work surface

System repair

Inspect devices, prepare rollback context, reinstall a broken GPU, audio, or network driver, and validate the result.

Project migration

Replace a BaaS or provider in a real codebase, rewrite config, run smoke checks, and summarize the diff.

Machine bootstrap

Take a new machine from blank state to working developer environment through one executable and one task log.

Support operations

Collect logs, config snapshots, and runtime state into support bundles that are usable by another human.

Local safety

Keep secrets outside the transcript, route risky actions through approvals, and avoid raw privileged prompt execution.

Operator clarity

Make it obvious what changed, what was approved, what version executed it, and how to roll back when possible.

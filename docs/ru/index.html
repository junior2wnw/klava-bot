Один executable. Один task log. Одна approval model.

Desktop agent, который должен не просто говорить про компьютер, а реально с ним работать.

Klava — это local-first Windows desktop agent для серьёзной работы: task execution, system inspection, guarded shell operations, secure onboarding и typed computer workflows, которые могут вырасти в repair, migration и support operations.

Что делает его другим

Klava — это не ещё одна terminal toy.

Цель — coherent local operator: система, которая может изучить машину, запросить approval, выполнить typed workflow, зафиксировать изменения и оставить материал, который сможет проверить другой инженер.

Workflow surface

Примеры того, как должен выглядеть operating model

System repair

Проверить устройства, подготовить rollback context, переустановить сломанный драйвер и провалидировать результат.

Project migration

Заменить BaaS или provider в реальном коде, переписать config, прогнать smoke checks и собрать diff summary.

Machine bootstrap

Привести новую машину от blank state к рабочему dev environment через один executable и один task log.

Support operations

Собрать logs, config snapshots и runtime state в support bundles, которыми реально сможет пользоваться другой человек.

Local safety

Держать secrets вне transcript, маршрутизировать рискованные действия через approvals и не давать модели raw privileged shell.

Operator clarity

Делать прозрачным, что поменялось, что approved, какая версия это выполнила и как откатить изменения.
